Technical overview

Interep consists mainly of four components:

  • Web application (reputation-service): A simple web app to allow users to join or leave groups supported by Interep.
  • APIs (reputation-service, subgraph): A set of APIs for interacting with Interep. Developers can use them to integrate groups into their applications.
  • Smart contracts (contracts): A set of Solidity smart contracts used to manage the onchain Interep groups.
  • JS libraries (interep.js): A monorepo of JavaScript libraries, some used by our backend, and others created to make Interep integration easier.

Interep is a system in which reputation, defined as recognition by other people of some characteristic or ability, plays a key role. In terms of social media reputation, though, reputation must be calculated on the basis of certain parameters and criteria. Let's first see how Interep calculates it in the next section.
